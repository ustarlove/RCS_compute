> 基于项目 `0511_mm_1` 源码整理，用于学习矩量法中网格和基函数两个核心模块。

## 一、网格生成（Mesh Generation）

### 1.1 为什么选正二十面体

对闭合 PEC 曲面（球面）做 MoM 离散，网格需要**覆盖均匀**且**全是三角形**。正二十面体是最接近球面的柏拉图立体，通过递归细分可以逼近任意精度的球面三角形网格（icosphere 方案）。

相比经纬线网格，icosphere 的三角形面积更均匀，不会在两极处出现过密的小三角形。

### 1.2 四个步骤

```
正二十面体顶点 → 20 个三角面 → 递归细分 → 归一化投影到球面
```

#### 步骤 1：生成 12 个顶点

```cpp
// mesh.cpp:49-66
double phi = (1.0 + sqrt(5.0)) / 2.0;  // 黄金比例 φ ≈ 1.618
double a = 1.0;
double b = 1.0 / phi;                   // ≈ 0.618
```

利用黄金比例构建 12 个顶点，坐标在 `(±a, ±b, 0)`、`(±b, 0, ±a)`、`(0, ±a, ±b)` 三个平面上的排列组合（共 12 个）。这些点内接于单位球面。

#### 步骤 2：生成 20 个三角形

`generate_icosahedron_triangles()` 中硬编码了 20 组顶点三元组，构成正二十面体的 20 个等边三角形面。

#### 步骤 3：递归细分（subdivide）

```
        v0                         v0
        /\                         /\
       /  \         →            /__\
      /    \                    /\  /\
     /______\                  /__\/__\
   v1        v2              v1   mid   v2

   1 个三角形 → 4 个三角形
```

对每个三角形的三条边分别取中点，用边中点连接，把原三角形分成 4 个小三角形。

**边的唯一标识**：用两个顶点索引组合为 64 位 key（高 32 位存较小索引，低 32 位存较大索引）。相邻三角形共享同一条边，其中点只创建一次，这样保证了**相邻三角形在细分后仍共点**。

每轮细分三角形数量变为原来的 4 倍：
| 细分次数 | 顶点数 | 三角形数 |
|---------|--------|----------|
| 0 | 12 | 20 |
| 1 | 42 | 80 |
| 2 | 162 | 320 |
| 3 | 642 | 1280 |

#### 步骤 4：归一化投影

```cpp
// mesh.cpp:150-158
double len = sqrt(v.x² + v.y² + v.z²);
v.x = v.x / len * radius;
v.y = v.y / len * radius;
v.z = v.z / len * radius;
```

每个顶点沿径向投影到半径为 `radius` 的球面上。细分时产生的新顶点在面上、不在球面上，必须归一化才能得到球面网格。

### 1.3 关键数据结构

```cpp
// mesh.h
struct Vertex { double x, y, z; };

struct Triangle {
    int v0, v1, v2;   // 顶点索引（指向 Mesh.vertices）
    int idx;           // 三角形编号
};

struct Mesh {
    vector<Vertex>  vertices;     // 顶点列表
    vector<Triangle> triangles;   // 三角形列表
};
```

### 1.4 三角形辅助方法

| 方法 | 计算方式 | 用途 |
|------|---------|------|
| `area()` | `½|AB × AC|` | RWG 系数、Gauss 积分权重 |
| `center()` | `(A+B+C)/3` | 奇异点处理的近似点 |
| `normal()` | `(AB×AC) / |AB×AC|` | 方向判断、物理解释 |

## 二、RWG 基函数（Rao-Wilton-Glisson）

### 2.1 核心概念

RWG 基函数定义在一条**公共边**上，涉及该边相邻的**两个三角形**。每条内部边对应一个基函数 fₙ(r)，是矩量法的未知量（展开系数 Iₙ）。

```
           apex⁺ (T⁺ 中与边相对的顶点)
            o
           /|\
          / | \    ← T⁺ (正三角形)
         /  |  \
     v1 o---|----o v2  公共边
         \  |  /
          \ | /    ← T⁻ (负三角形)
           \|/
            o
           apex⁻ (T⁻ 中与边相对的顶点)

    f(r) 的方向：从 T⁺ 流向 T⁻（跨过公共边）
```

### 2.2 构建流程（`RWGData::build`）

#### 第一步：建立边到三角形的映射

```cpp
// rwg.cpp:7-27
unordered_map<long long, pair<int, int>> edge_tri_map;
```

遍历所有三角形的所有三条边，同样用 64 位 key 标识边。每个 key 对应两个值：
- `first`：第一个遇到该边的三角形
- `second`：第二个遇到该边的三角形（`-1` 表示未找到）

#### 第二步：筛选公共边

```cpp
// rwg.cpp:36-78
if (t2 == -1) continue;  // 跳过边界边
```

只保留 `t2 != -1` 的边，即被两个三角形共享的内部边。对封闭球面来说所有边都是内部边（没有边界）。

**未知量数量**：对于封闭三角网格，满足欧拉公式 `V - E + F = 2`，每条边两个三角形、每个三角形三条边 → `E ≈ 1.5F`，即 RWG 基函数个数 ≈ 三角形数 × 1.5。

#### 第三步：计算 RWG 系数

```cpp
rwg_coeff1 = l / (2 × A⁺)   // T⁺ 中的系数
rwg_coeff2 = l / (2 × A⁻)   // T⁻ 中的系数（取绝对值，负号在使用时加）
```

其中 `l` 是公共边长，`A⁺`/`A⁻` 是两个三角形的面积。两个系数都是正数，符号由调用时的上下文决定。

#### 第四步：找 apex 顶点

```cpp
// rwg.cpp:53-61
auto find_apex = [&](int tri_idx, int ev0, int ev1) -> Vertex {
    // 找到三角形中既不是 ev0 也不是 ev1 的那个顶点
};
```

Apex 是三角形中**与公共边相对的顶点**，即不属于这条边的那个顶点。T⁺ 和 T⁻ 各有一个 apex，它们不同。

### 2.3 基函数求值（`get_rwg`）

基函数 fₙ(r) 的定义：

```
在 T⁺ 上：f⁺(r) = +l/(2A⁺) · (r - apex⁺)
在 T⁻ 上：f⁻(r) = -l/(2A⁻) · (r - apex⁻)
在其他三角形上：f(r) = 0  ← 紧支撑性
```

代码实现（`rwg.cpp:83-106`）：

```cpp
if (tri_idx == e.t1) {
    coeff = +rwg_coeff1;    // +l/(2A⁺)
    apex  = e.apex1;
} else if (tri_idx == e.t2) {
    coeff = -rwg_coeff2;    // -l/(2A⁻)
    apex  = e.apex2;
}
// f(r) = coeff × (r - apex)
```

**关键性质**：
- **紧支撑**：每个 fₙ 只在两个三角形上非零
- **法向连续性**：fₙ 在公共边上的法向分量连续（等于边长 l，归一化后为 1）
- **线性分布**：从 apex 到公共边，幅度线性减小（在 apex 处为 0，在边上最大）

### 2.4 散度计算（`get_divergence`）

基函数的表面散度在三角形内为**常数**：

```
∇ₛ·f⁺ = +l/A⁺     （电荷"流出"T⁺）
∇ₛ·f⁻ = -l/A⁻     （电荷"流入"T⁻）
```

推导：在二维面上 ∇ₛ·r = 2（被限制在面内），所以：
```
∇ₛ·[l/(2A⁺)·(r - apex)] = l/(2A⁺)·2 = l/A⁺
```

代码实现（`rwg.cpp:108-117`）：

```cpp
if (tri_idx == edges[edge_idx].t1) {
    return 2.0 * rwg_coeff1;     // 2 × l/(2A⁺) = l/A⁺
} else if (tri_idx == edges[edge_idx].t2) {
    return -2.0 * rwg_coeff2;    // -l/A⁻
}
```

**电荷守恒**：在整个基函数对上积分：

```
∫ ∇ₛ·f⁺ dS⁺ + ∫ ∇ₛ·f⁻ dS⁻ = (l/A⁺)·A⁺ + (-l/A⁻)·A⁻ = l - l = 0
```

这就是 RWG 基函数的精髓——**每个基函数对自动满足电荷连续性**，无需额外约束。

### 2.5 物理图像：从电路类比理解

把 fₙ 看作**支路电流的参考方向**：
- fₙ 规定电流从 T⁺ 流向 T⁻ 的**空间分布形状**
- 实际的电流大小和方向由解出的复系数 Iₙ 决定
- Iₙ > 0 → 真实电流与 fₙ 同向；Iₙ < 0 → 反向

就像电路分析中先假设好支路电流方向再解方程，解出负值就说明实际方向相反。

## 三、网格与基函数的衔接关系

```
Mesh ──生成──→ vertices[], triangles[]
                   │
                   ▼
RWGData::build ───→ edges[]  (每条边 = 一对相邻三角形)
                    rwg_coeff1/2[]  (l/2A)
                    apex1/2[]  (对顶点)
                   │
                   ▼
              get_rwg(边, 三角形, 位置) → f(r) 向量值
              get_divergence(边, 三角形) → ∇·f 标量值
```

这两个模块构成了 MoM 的前半段流水线，为后续的 EFIE 矩阵填充（阻抗矩阵 Z 和激励向量 b）提供几何信息。
