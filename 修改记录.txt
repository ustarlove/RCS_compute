FMM 加速方案实施计划

 背景

 当前 MoM+EFIE 代码使用 O(N²) 稠密阻抗矩阵和 O(N²) 矩阵-向量积。本项目将添加单层快速多极子 (Single-Level FMM)
 加速，使用平面波展开（diagonal form），将远场相互作用从 O(N²) 降为 O(N^1.5)。

 核心算法

 EFIE 核的 4 通道分解

 EFIE 矩阵元：Z_ij = jkη ∫∫ [f_i·f_j - k⁻²(∇·f_i)(∇·f_j)] G(R) dS' dS

 对远场，做中心点近似，G(R) 用平面波展开（对角形式），分解为 4 个独立标量通道：
 - 源权重 s_j：(f_jx, f_jy, f_jz, div_j/k)
 - 测试权重 t_i：(f_ix, f_iy, f_iz, -div_i/k)

 每次迭代的 FMM 矩阵-向量积

 1. 聚合 (Aggregation)：每个盒子对所有 K 个方向求和源贡献
 2. 转移 (Translation)：非空盒子对之间施加对角转移算子 T_L
 3. 解聚 (Disaggregation)：在每个测试函数处汇总各方向的入射场
 4. 加近场：CSR 稀疏矩阵-向量积

 转移算子

 T_L(kR, cosθ) = Σ_{l=0}^{L} (-j)^l (2l+1) h_l^{(2)}(kR) P_l(cosθ)

 通过球 Hankel 函数和 Legendre 多项式的递推关系计算。

 新增文件

 1. vec3_ops.h — 向量运算工具（纯 inline）

 Vertex 的运算符重载和工具函数：+, -, *scalar, dot, cross, norm, normalize

 2. fmm_translator.h / fmm_translator.cpp — 转移算子 & 角谱积分

 - PWSDirection 结构体（k̂ 方向 + 权重）
 - Translator 类：
   - generate_quadrature(L) — Gauss-Legendre (cosθ) × 均匀 (φ) 球面采样
   - compute(kR, cos_theta, L) — 计算 T_L
   - hankel_h2() / legendre_p() — 递推实现

 3. fmm.h / fmm.cpp — FMM 核心

 - FMMBox 结构体（网格索引、中心、基函数列表、近场/远场盒子列表）
 - FMM 类：
   - build_boxes() — 构建 3D 规则网格，分配基函数到盒子
   - precompute_patterns() — 为所有基函数预计算 4 通道辐射/接收方向图
   - fill_nearfield(efie) — 用精确 EFIE 双重积分填充 CSR 近场矩阵
   - matvec(x, y) — FMM 加速的矩阵-向量积
   - get_matvec_functor() — 返回 std::function<void(vec,vec)> 供求解器使用

 关键设计决策：
 - 盒子大小 = 0.3λ（可配置）
 - 截断数 L：L ≈ kD/2 + 1.8 (kD/2)^{1/3}，D/2 为盒子半宽
 - 近场：±1 盒子范围内的所有盒子对（最多 27 邻域）
 - 辐射方向图用中心点近似（三角形重心处求 RWG 值 × 面积）
 - 基函数"中心"取其两个三角形重心的平均值
 - 转移矩阵在初始化时预计算（存入扁平数组），迭代时只做乘加

 修改文件

 4. efie.h / efie.cpp

 - compute_interaction(int i, int j) 从 private 改为 public const
 - 使 FMM 的近场填充能调用精确双重积分

 5. cg_solver.h / cg_solver.cpp

 - 新增 solve_bicgstab_functor(matvec_functor, b, x, tol, max_iter) 静态方法
 - 用 std::function<void(const vector&, vector&)> 替代稠密 Z 矩阵
 - 原有 solve_bicgstab(Z, b, x) 保持不变，便于对比

 6. main.cpp

 - 添加 use_fmm 开关
 - FMM 路径：build_boxes() → precompute_patterns() → fill_nearfield() → solve_bicgstab_functor()
 - 可选：同时用稠密法求解并对比结果

 7. Makefile

 - SRCS 添加 fmm.cpp fmm_translator.cpp

  【2026-06-17】FMM → MLFMA 改造（双立方插值）
══════════════════════════════════════════════════════════

  背景
  ────
  将原有的单层 FMM (Single-Level FMM) 升级为多层快速多极子算法 (MLFMA)，
  层级间方向图传递采用双立方 (bicubic) Lagrange 插值。

  核心改动
  ────────
  1. 八叉树结构
     - 替代单层均匀网格，构建多层空间划分
     - 层 0 (最粗, box=1.2m) → 层 1 (box=0.6m) → 层 2 (最细, box=0.3m)
     - 自底向上通过中心点坐标建立父子关系
     - 每层独立计算截断数 L 和角采样方向数 K

  2. 互作用列表 (Interaction List)
     - 标准 MLFMA 定义: 父层近邻盒子的所有子盒子，排除本层近邻
     - 最粗层特殊处理: 直接取所有其它非空且非近邻盒子
     - 最细层使用 child_boxes 时修正为 parent.child_boxes（修复最细层无子层的 bug）

  3. 双立方插值 / 反插值
     - θ 方向: 4 点 Lagrange 插值在 Gauss-Legendre 非均匀节点
     - φ 方向: 4 点 Lagrange 插值在均匀周期节点（循环取模）
     - 插值 (interp_bicubic): 细层 K → 粗层 K（增加角采样）
     - 反插值 (anterp_bicubic): 粗层 K → 细层 K（减少角采样，插值算子的伴随）

  4. matvec 四阶段流水线
     阶段 1: 近场 CSR matvec（仅最细层）
     阶段 2: 上行聚合 — 最细层聚合 → [相位平移 + 双立方插值] → 父层累加 → ... → 最粗层
     阶段 3: M2L 转移 — 每层互作用列表内对角转移算子 T_L
     阶段 4: 下行解聚 — 最粗层 → [双立方反插值 + 相位平移] → 子层分发 → ... → 最细层 → 基函数

  新增文件
  ────────
  - mlfma.h     — MLFMA 类声明（八叉树层级、LevelData、双立方插值接口）
  - mlfma.cpp   — MLFMA 完整实现（~780 行）

  修改文件
  ────────
  - main.cpp    — FMM → MLFMA（include、变量名、调用接口替换）
  - Makefile    — SRCS 添加 mlfma.cpp

  保留文件
  ────────
  - fmm.h / fmm.cpp             — 原单层 FMM 代码保留，供参考对比
  - fmm_translator.h / fmm.cpp  — Translator 和 PWSDirection 被 MLFMA 复用

  关键技术细节
  ────────────
  - 截断数 L: L ≈ ka + 1.8·(ka)^{1/3}·log10(1/ε)，每层独立计算，上限 20
  - 方向数 K: K = 2·(L+1)²（Gauss-Legendre θ × 均匀 φ 张量积）
  - 相位平移: dst[p] = src[p] · exp(j·k₀·k̂_p · (r_old - r_new))
  - 常数 C = k₀²·η / (16π²)

  运行结果
  ────────
  - N = 480 未知量，3 层八叉树
  - 最细层: 41 非空盒子, K=128, 1190 M2L pairs
  - 中间层: 8 非空盒子, K=288, 0 M2L pairs (全近邻)
  - 最粗层: 1 非空盒子, K=578, 0 M2L pairs
  - GMRES(30) 104 次迭代收敛，|r|/|b| = 7.5e-7
  - RCS 输出合理

  当前限制
  ────────
  - N=480 规模下粗层 M2L 对数为零，多层级优势未体现
  - 增大 refinement 参数至 3~4（N≈2000~8000）后粗层将产生有效 M2L 互作用
  - 单层 FMM 代码保留可对比验证
